# QA — ручной чек-лист

Автотестов в проекте нет (см. «Почему без автотестов» ниже) — качество
подтверждается `npm run typecheck`, `npm run build` и этим чек-листом перед
каждым релизом (мерджем в `main`, который автодеплоит на GitHub Pages).

## Перед каждым PR / мерджем в `main`

- [ ] `npm run typecheck` — чисто, без ошибок и `any`.
- [ ] `npm run build` — собирается без ошибок и предупреждений о размере бандла.
- [ ] `npm run preview` — открыть собранный `dist/` и глазами пройти по всем секциям.
- [ ] В консоли браузера нет ошибок/предупреждений (React, GSAP, 404 на ассеты).
- [ ] Нет горизонтального скролла ни на одном брейкпоинте (см. ниже).

## Брейкпоинты

Обязательные контрольные точки — тестировать вручную (DevTools → Device Toolbar
или реальные устройства):

| Ширина | Что проверять |
|---|---|
| 390px | базовый мобильный (iPhone 12/13/14). Худший случай для переполнений. |
| 430px | крупный телефон (iPhone Pro Max, большинство Android). |
| 768px | планшет портрет. Навбар обычно уже переключается на десктопный. |
| 1024px | планшет альбом / малый ноутбук. |
| 1440px | десктоп, основная референсная ширина дизайна. |
| 1920px | широкий десктоп — контент не должен «расползаться» бесконтрольно. |

Для каждой ширины:
- [ ] Нет горизонтального скролла (`document.documentElement.scrollWidth <= innerWidth`).
- [ ] Текст не обрезается и не наезжает сам на себя.
- [ ] Тач-таргеты (кнопки, ссылки, бургер) не меньше ~44×44px на мобилке.
- [ ] На десктопе ни одна интерактивная цель не меньше 24×24px (WCAG 2.2 AA) —
      включая тонкие засечки DepthRail и рейла эр: у них увеличена зона нажатия,
      а не сама линия.

Быстрая проверка первых трёх пунктов одним куском в консоли:

```js
const vw = document.documentElement.clientWidth
console.table({
  overflow: document.documentElement.scrollWidth > vw ? 'ЕСТЬ' : 'нет',
  smallTargets: [...document.querySelectorAll('button, a[href], [role="button"]')]
    .filter(el => !el.closest('[inert]') && !el.classList.contains('sr-only'))
    .filter(el => { const r = el.getBoundingClientRect(); return r.width && r.height < 44 })
    .length,
})
```

## Функциональность

- [ ] **Hero**: на десктопе спотлайт-раскрытие видео следует за курсором; на
      touch-устройстве без гироскопа — плавно дрейфует по кругу.
  - [ ] На мобилке видео `object-cover`, без чёрных полос и без обрезания важной части кадра.
  - [ ] Если устройство поддерживает `deviceorientation` — лёгкий гиро-параллакс видео.
- [ ] **Навигация**: клик по пунктам навбара плавно скроллит к нужной секции с учётом отступа шапки.
- [ ] **DepthRail** (десктоп ≥ `lg`): точки подсвечивают активную секцию при скролле, клик — скроллит к ней.
- [ ] **Меню-шахта** (одинаково на всех ширинах):
  - [ ] Открывается по «МЕНЮ», закрывается по крестику и по `Escape`; после `Escape` фокус возвращается на кнопку «МЕНЮ».
  - [ ] Фоновый скролл блокируется, пока меню открыто, и разблокируется при закрытии.
  - [ ] Фокус визуально виден на всех пунктах при табуляции.
  - [ ] **Ловушка фокуса**: Tab с последнего пункта уводит на первый, Shift+Tab с первого — на последний; фокус не уходит на страницу за меню.
  - [ ] **Закрытое меню не ловит Tab**: пройти табом всю страницу — ни один пункт меню не получает фокус (обеспечено `inert`).
- [ ] **Переключатель языка (RU/UZ)**:
  - [ ] Меняет весь видимый текст без перезагрузки страницы.
  - [ ] `<html lang>` синхронизируется с выбором.
  - [ ] Выбор сохраняется в `localStorage` и восстанавливается после перезагрузки.
- [ ] **Счётчики (Stats)**: анимируются один раз при попадании во вьюпорт, не запускаются повторно при обратном скролле.
- [ ] **CTA**: «Начать спуск» / основной CTA всегда ведёт к секции `#descent`; ссылка на Telegram открывается в новой вкладке (`target="_blank"`, `rel="noopener noreferrer"`).
- [ ] **Курсор**: на fine-pointer устройствах кастомный курсор меняет режим над медиа (лупа) и над кнопками (подпись); нативный курсор скрыт только при активном кастомном.
  - [ ] В секции Voice (светлый фон) курсор, пыль и осколки перекрашиваются в тёмный и остаются видимыми.
- [ ] **Expeditions / Footer**: при наведении на строку фото места летит за курсором с отставанием и не выходит за края экрана; уход мышью с списка убирает превью.
- [ ] **Досье маршрута**: при раскрытии профиль хребта рисуется линией слева направо, а не появляется целиком.
- [ ] **Descent**: отсчёт глубины доходит ровно до −4600 М и не «проскакивает» дальше.

## Мобильные жесты (реальное устройство, не эмулятор)

Эмулятор DevTools не даёт ни гироскопа, ни вибрации — этот блок проверяется
только на телефоне.

- [ ] **Hero, долгое касание**: удержание пальца сужает луч; отпускание возвращает.
- [ ] **Hero, двойной тап**: одна вспышка на камне, без залипания.
- [ ] **Гироскоп**: наклон телефона двигает луч, фото эры и разворачивает карточки архива; в шапке появляется «КРЕН ±N°».
- [ ] **Вибрация**: короткий отклик ровно на трёх событиях — смена эры, снап карточки архива, открытие меню. Больше нигде.
- [ ] **Скролл пальцем** ощущается нативным (без «резинового» подхватывания Lenis).
- [ ] **Полка архива**: свайп листает со снапом, счётчик и имя образца следуют за текущим ящиком.
- [ ] **Маршруты и ссылки футера**: подсветка сама переходит на элемент в центре экрана при скролле.
- [ ] **Descent** занимает экран целиком, обе кнопки достаются большим пальцем.

## Авто-деградация перфа

- [ ] На слабом устройстве (или при троттлинге CPU 6×) через ~2 с после загрузки
      на `<html>` появляется класс `perf-plain`, пропадают зерно, пыль и
      backdrop-blur — при этом весь текст, лейаут и навигация на месте.
- [ ] При `prefers-reduced-motion` класс ставится сразу, без замера.

## Доступность (a11y)

- [ ] Skip-ссылка «Перейти к содержимому» появляется по Tab первым элементом и ведёт к первой секции.
- [ ] Один `<h1>` на странице (Hero), заголовки секций — `<h2>`/`<h3>` по иерархии.
- [ ] Все интерактивные иконки (бургер, закрыть меню, соцссылки) имеют `aria-label`.
- [ ] Видимый keyboard-focus на всех интерактивных элементах (кнопки, ссылки, тумблер языка, пункты меню).
- [ ] Полная навигация с клавиатуры (Tab/Shift+Tab/Enter/Escape) без «ловушек» фокуса.
- [ ] `prefers-reduced-motion: reduce` (эмуляция в DevTools → Rendering):
  - [ ] Кастомный курсор не монтируется.
  - [ ] Параллакс/reveal-анимации отключены, контент виден сразу.
  - [ ] Marquee (бегущая строка терминов) не крутится.

## Локализация

- [ ] Переключение RU → UZ → RU не оставляет «залипших» английских/русских строк на узбекской версии.
- [ ] Числа (Stats) форматируются согласованно в обеих локалях.
- [ ] Апострофы в узбекской латинице (`oʻ`, `gʻ`) отображаются корректно во всех использованных шрифтах.

## Производительность

- [ ] Lighthouse (Chrome DevTools, режим Mobile) на собранном `dist/` через `npm run preview`:
  - [ ] Performance ≥ 85, LCP < 2.5s на симулированном 4G.
  - [ ] Нет layout shift от поздней загрузки шрифтов/изображений (CLS ≈ 0).
- [ ] Вкладка Network: `base.webp` и `reveal.mp4` не блокируют первый рендер текста.
- [ ] Canvas-эффекты (частицы, спотлайт) не крутятся, когда их секция вне вьюпорта (проверить через Performance/Task Manager — CPU падает при скролле мимо).

## Кросс-браузерность

- [ ] Chrome/Edge (Chromium) — десктоп и мобильная эмуляция.
- [ ] Firefox — desktop.
- [ ] Safari (или WebKit) — особое внимание к `backdrop-filter`, `mask-image`, видео `autoplay muted playsinline`.

## Почему без автотестов

Проект — статический одностраничный лендинг без бизнес-логики, форм и внешних
интеграций: риск регрессий сосредоточен в вёрстке, анимациях и визуальном
восприятии, а не в вычислениях. Автотесты (Vitest/Playwright) добавлять
осмысленно, если появится: логика с ветвлениями для юнит-тестов, или бюджет на
поддержку visual-regression инфраструктуры для анимаций. До этого момента
ручной чек-лист даёт больше сигнала за меньшую цену.
